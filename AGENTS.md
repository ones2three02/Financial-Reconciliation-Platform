# AGENTS.md — 全局开发指南（通用原则）

## 0. 适用范围

本文件是全局默认规则，适用于所有项目。项目根目录或子目录存在自己的 AGENTS.md 时，以更具体
的规则为准（就近覆盖），未被覆盖的部分仍然生效。

---

## 1. 核心原则
- 高效：优先解决实际问题，避免过度流程化。
- 稳定：默认最小变更，不破坏现有功能。
- 准确：先理解上下文，再修改代码；结论必须有依据，不臆测。
- 可维护：结构清晰、职责分离、可测试、可回滚。
- 中文化：沟通说明、注释、文档、提交信息默认使用中文；**代码标识符（变量名、函数名、类名）使用英文**。

---

## 2. 工作方式
1. **理解需求**：先读相关代码、配置、报错和上下文。**若项目根目录下存在 `.ai-organization/`，优先加载其中的组织政策、部门岗位定义与 Skills。**
2. **明确范围**：只改和当前任务直接相关的内容。
3. **制定计划**：简单直接处理；复杂任务先列简短步骤。**在组织化项目中，必须依照 `templates/project-raci.md` 定义分工，并通过 `skills/organization-manager/SKILL.md` 校验能力缺口，严禁单 Agent 越权决策或独立执行复杂业务闭环。**
4. **验证结果**：优先运行构建、测试或静态检查。
5. **总结交付**：按交付模板进行总结。

---

## 十、AI 组织化协同规范（若项目包含 .ai-organization）
当且仅当项目根目录存在 `.ai-organization/` 目录时，必须强制激活以下规则：
1. **角色与身份契合**：Agent 是拥有特定职责的“组织成员”。开发时必须根据 `registry/agent-registry.yaml` 明确自身岗位 ID，并在 `templates/agent-job-description.md` 定义的授权边界内活动。
2. **能力缺口与招聘**：遇到当前活跃 Agent 均不持有的能力时，禁止“假装”拥有该技能。必须通过 AI HR 经理生成 `templates/job-request.md` 申请编制并运行招聘及面试流程。
3. **人机升级协作 (Escalation)**：在涉及财务预算、法务合规、敏感企业制度变更或死锁冲突时，必须挂起任务并生成 `human-decision-required.md`，升级至 L4 级（人类专家）终审签字。
4. **决策投票与会签**：重大架构调整或数据库 Schema 变更，必须按照 `core/decision-policy.md` 执行多部门会签投票，并在 `templates/decision-record.md` 中留下共识归档。
