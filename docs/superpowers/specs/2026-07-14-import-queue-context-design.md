# 文件导入独立队列设计

## 背景

当前导入中心只有一个全局文件队列。用户在美团模板下选择文件后切换到抖音模板，队列仍保留原美团文件；导入按钮又只检查队列非空，于是系统会尝试用抖音模板预检美团文件。

后端模板白名单能够阻止错误文件落库，但前端仍允许发起明显错误的操作，容易造成困惑和重复劳动。

## 目标

- 每份待导入文件永久绑定选择文件时的账期、模板和文件级门店上下文。
- 切换模板或门店只切换可见队列，不改变既有文件的归属。
- 已完成、内容重复或待确认门店的文件不能再次被“开始导入”处理。
- 导入过程使用不可变上下文，界面状态变化不能改变已经开始的请求归属。

## 方案选择

采用 B+ 独立队列方案：

- 渠道文件按模板分别保存队列。
- 门店财务表按“模板 + 标准门店”分别保存队列。
- 切换账期时清空全部临时队列，避免跨账期复用。

不采用切换模板即清空队列，因为财务需要在多个来源之间来回切换查看导入进度；也不采用保留单一队列再弹警告，因为文件仍可能被错误模板处理。

## 状态模型

### 队列上下文

队列上下文包含：

```text
businessDate
profileCode
storeId
```

渠道模板的 `storeId` 固定为空。门店财务表必须包含标准门店 ID。

上下文键规则：

```text
渠道模板：businessDate + profileCode
门店财务表：businessDate + profileCode + storeId
```

每个队列项保存自己的上下文快照，不能在切换模板或门店后被改写。

### 可执行状态

只有以下状态可以进入本次导入：

- `ready`
- `failed`（允许修正上下文或文件后重试）

以下状态只用于展示结果，不得重复发送导入请求：

- `imported`
- `duplicate`
- `attention`

按钮显示可执行文件数量，而不是当前队列总数。没有可执行文件时按钮禁用。

## 页面行为

### 切换模板

- 显示目标模板自己的队列。
- 原模板队列保留，切回后继续显示原文件及结果。
- 模板卡片显示该模板的待处理数、失败数和已完成数。

### 切换门店

- 仅门店财务表按门店隔离队列。
- 切换门店后显示该门店自己的文件，不复用其他门店文件。

### 切换账期

- 清空全部临时队列和页面提示。
- 重新加载目标账期批次。
- 已经写入数据库的导入历史不受影响。

### 导入过程

- 开始时固定批次 ID、账期、模板、门店和待执行文件列表。
- 处理期间禁用模板和门店选择。
- 每个请求始终使用队列项绑定的上下文，不读取循环过程中可能变化的界面值。
- 完成后仅在页面仍处于原账期时刷新当前批次；若用户已经切换账期，不覆盖新账期页面状态。

### 清空列表

“清空列表”只清空当前上下文队列，不影响其他模板或门店队列。

## 实现边界

- 将队列键、上下文匹配、可执行状态统计提取为纯 TypeScript 模块。
- `ImportCenter.vue` 只负责将界面选择映射到当前上下文，并调用纯状态函数。
- 不新增前端依赖；使用 Node 22 内置测试运行器执行 TypeScript 单元测试。

## 测试与验收

- 美团队列在切换到抖音后不可见，抖音导入按钮不处理美团文件。
- 切回美团后仍显示原美团文件及其导入状态。
- 门店财务表在两个门店之间切换时分别保留各自队列。
- 切换账期后所有临时队列清空。
- 已导入文件保留展示，但可执行数量为零且不会重复提交。
- 导入开始后即使界面状态发生变化，请求仍使用开始时绑定的上下文。
- 前端 TypeScript 单元测试和生产构建通过。

## 非目标

- 不在浏览器刷新后恢复临时队列。
- 不把待导入文件内容写入 localStorage 或数据库。
- 不改变后端模板校验和文件内容判重规则。
