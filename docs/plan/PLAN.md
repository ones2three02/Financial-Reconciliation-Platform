# 总体开发计划 (PLAN.md)

> 2026-07-14 起的规范驱动增量开发以 [执行路线](./EXECUTION_2026-07-14.md) 和
> [后端正确性基础计划](./BACKEND_FOUNDATION_2026-07-14.md) 为当前执行依据；本文以下内容保留为原始阶段概览。

FRP 项目采用计划驱动开发模式，整体分为八个开发阶段。每个阶段有明确的目标与验收标准。

## 阶段规划

### Phase 1：项目初始化
* **目标**：搭建前后端框架（FastAPI & Vue3）、配置 MySQL/SQLite 数据库连接、设计核心配置表。
* **验收标准**：前后端服务可正常启动；完成数据库迁移与连接测试。

### Phase 2：文件导入
* **目标**：实现拖拽及批量文件上传至后端暂存。
* **验收标准**：前端支持多文件拖拽；后端可正确接收并根据所选分类标记文件来源。

### Phase 3：Excel 解析
* **目标**：基于 Pandas 动态读取 Sheet，自动提取行键值对。
* **验收标准**：支持动态识别未知列头，自动持久化原始数据为 JSON。

### Phase 4：数据清洗
* **目标**：对原始金额（去符号/千分位）、日期格式统一为 YYYY-MM-DD，剔除空行。
* **验收标准**：清洗后数据输出为统一 Schema；保存清洗日志，原始数据不被修改。

### Phase 5：门店标准化
* **目标**：通过建立别名映射库，将非标准门店名（如“杨一一游泳馆”）匹配至标准门店。
* **验收标准**：捕获未匹配门店并拦截进入待认领界面；财务确认映射关系后永久生效。

### Phase 6：自动对账
* **目标**：执行对账计算公式，并自动按日期+标准门店输出核对差额。
* **验收标准**：自动算出差异；标记“一致”、“有差异”与“缺失数据”三种状态。

### Phase 7：Dashboard
* **目标**：提供财务大屏看板，呈现汇总数据。
* **验收标准**：图形化展示今日对账门店比例、异常门店趋势与差异大户排行。

### Phase 8：异常管理
* **目标**：对异常差异门店进行全生命周期管理。
* **验收标准**：支持添加备注原因，手动标记解决。
