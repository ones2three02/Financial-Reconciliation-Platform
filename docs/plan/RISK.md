# 风险分析与应对策略 (RISK.md)

### 1. Excel 格式突变风险
* **风险描述**：第三方导出模板的列头名称忽然变更，导致解析引擎找不到关键列。
* **应对策略**：采用“字段映射配置”。一旦报错，财务人员在“字段映射配置”页将新列名关联到标准字段，无需修改代码即可恢复对账。

### 2. 门店重名与多义风险
* **风险描述**：例如“天门店”和“天门二店”被错误归并，或新开“杨一一旗舰店”未关联导致账目缺失。
* **应对策略**：强制启用未知别名拦截机制。凡是清洗引擎碰到的未知门店别名，一律归为 `pending` 状态且对账结果标为 `missing_data`，促使财务在“门店标准化”中心进行人工核认与绑定。

### 3. 数据重刷与计算覆盖风险
* **风险描述**：重新清洗或对账可能会覆盖已人工核实并备注的异常条目。
* **应对策略**：在 ReconciliationResult 表中，如果记录的 status 不是 consistent 且 `is_resolved` 为 true，重刷对账数据时将保留用户的 `remarks`、`resolved_by` 等核实痕迹。
