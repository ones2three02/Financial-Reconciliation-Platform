# 企业级安全基础加固计划

## 目标

在前端批次工作流开发前，消除阶段性 code review 发现的阻断问题，确保身份、审计操作人、敏感配置和工作簿资源边界具备可上线演进的基础。

## 范围

1. 新增用户与服务端会话模型，使用标准库 `scrypt` 保存密码摘要、随机会话令牌只保存哈希；
2. 登录外的 API 默认要求 Bearer 会话，审计操作人从服务端身份获取，不接受客户端伪造；
3. 删除硬编码演示密码和固定令牌，取消应用导入时自动建库、自动写入门店；
4. 将已跟踪的 `backend/.env` 移出版本控制，提供无密钥的 `.env.example`；
5. CORS 改为环境白名单，禁用通配来源；
6. 为工作簿 ZIP 解压总量和压缩比设置上限，防止压缩炸弹；
7. 增加认证、越权、配置和恶意工作簿测试，完成全量回归。

## 非目标

- 不增加经办/复核双人审批；
- 不接入企业 SSO、LDAP 或外部 OAuth，本轮保留未来替换边界；
- 不执行生产部署、生产密钥轮换或生产数据库迁移；
- 不新增第三方依赖或升级依赖大版本。

## 数据库影响与回滚

- 增量迁移 `0003_authentication_foundation` 只新增 `app_user` 与 `user_session`；
- 回滚先撤销会话，再撤销用户表，不修改对账、导入和原始数据；
- 本地已有数据库仍必须先使用 `backup_sqlite.py` 备份；仓库当前无可迁移的本地 `.db` 文件，因此本轮只验证空库升级/降级。

## 验收标准

- 未认证请求不能访问业务 API；
- 登录返回随机令牌，数据库中不保存明文令牌或密码；
- 别名确认、零收入、关账、重开和导入的审计操作人来自会话用户；
- 仓库不再跟踪 `.env`，代码中无固定密码或固定访问令牌；
- 超限或高压缩比工作簿在 OpenPyXL 解析前被拒绝；
- 后端全量测试、迁移往返、编译和依赖检查全部通过。
