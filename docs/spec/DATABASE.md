# 数据库规范 (DATABASE.md)

系统数据库使用 MySQL 8（开发环境可降级使用 SQLite），采用四层数据管道模型。

## 数据模型四层架构
1. **ImportFile**：上传任务登记表。记录文件的上传状态、大小、原始文件名及归属的数据源通道。
2. **RawData**：原始数据镜像表。将 Excel 中的行列数据转换成 Key-Value 结构的 JSON 直接归档，保存最原始的状态，便于审计。
3. **CleanData**：标准化交易明细表。经过字段映射、金额转换、日期标准化、标准门店别名绑定后的每一笔标准账目流水。
4. **ReconciliationResult**：对账结果汇总表。按照对账日期 + 标准门店主键，对五大渠道金额进行汇总比对后的轧账明细。

## 表约束与外键
* 级联删除：`raw_data`、`clean_data` 必须关联 `import_file_id` 并配置 `ON DELETE CASCADE`，以防删除文件导入日志时残留垃圾数据。
